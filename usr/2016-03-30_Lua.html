原理: GBK是2字节编码，实现GBK编码对应utf-8字，使用lua的查表替换成utf-8编码即可 - 主要参考链接: [gbk编码表与编码字库](http://www.bo56.com/gbk%E7%BC%96%E7%A0%81%E8%A1%A8%E4%B8%8E%E7%BC%96%E7%A0%81%E5%AD%97%E5%BA%93/) ```text gbk编码分两部分，一部分是单字节编码，另一部分是双字节编码。 gbk编码中，前128个编码都是单字节编码。单字节编码从00-7F，与ASCII相对应。 在单字节编码之后就是双字节编码。第一个字节范围是81-FE。第二字节的一部分领域在40–7E，其他领域在80–FE。 这样可以通过第一个字节就可以判断是单字节编码还是双字节编码。 范围 第1字节 第2字节 编码数 字数 内容 GBK/1 A1–A9 A1–FE 846 717 GB2312非汉字符号 GBK/2 B0–F7 A1–FE 6,768 6,763 GB2312 汉字 GBK/3 81–A0 40–FE(7F除外) 6,080 6,080 扩充汉字 GBK/4 AA–FE 40–A0(7F除外) 8,160 8,160 扩充汉字 GBK/5 A8–A9 40–A0(7F除外) 192 166 扩充非汉字 用户定义 AA–AF A1–FE 564 用户定义 F8–FE A1–FE 658 用户定义 A1–A7 40–A0(7F除外) 672 合计: 23,940 21,886 ``` - 第一步: 生成对应表 ```lua -- 第一个字节范围是81-FE。第二字节的一部分领域在40–7E，其他领域在80–FE for aa=129, 254 do for bb=64, 254 do if bb ~= 127 then io.write("G", string.format("%X%X", aa, bb), "='", string.char(aa), string.char(bb), "',\n") end end end ``` - 第二步: 将生成的列表文件编码改成utf-8，这样就可以实现转utf-8，如果存为ucs-2，理论上可以实现gbk转ucs-2，呵呵 - 第三步: 将这个文件改后缀成.lua，改成以下形式的代码，就是将生成的内容替换tbgbk2utf8这个hash表中的内容，为什么在key名上加上G，是因为有8140这样的编码，lua会当成数组解释，很蛋疼，所以统一加上G，这样就成字符串了，然后就正常了 ```lua function g2u(t) local tbgbk2utf8 = { G8140='丂', G8141='丄', G8142='丅', GFEFE='' } return string.gsub(t, "([\128-\255=].)", function (c) return tbgbk2utf8[string.format("G%X%X", string.byte(c, 1, 2))] end) end ``` - 用法: 在lua中调用 g2u函数即可 ```lua utf8Str = g2u(GBKStr) ```lua
